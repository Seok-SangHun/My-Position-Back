<h2>6. 느낀점</h2>
<h3>6-1. 어려웠던 부분</h3>
<h4>📌 405오류 발생</h4>

프로젝트를 진행하면서 가장 어려웠던 부분 중 하나는 <strong>HTTP 405 오류를 해결</strong>하는 것이었습니다. 처음 405 오류를 접했을 때, "클라이언트가 요청한 메소드가 서버에서 지원되지 않을 때 발생합니다."라는 설명을 참고하여 GET과 POST 메소드가 올바르게 작성되었는지 확인했지만, 문제를 찾는 데 어려움을 겪었습니다. 오류 해결에 시간이 걸렸으나, <strong>로그를 분석하고 요청을 하나씩 출력</strong>해보니<strong> @RequestBody를 누락</strong>하여 발생하는 것을 알게 되었습니다.<br>

이 문제를 해결하기 위해 <strong>API 문서를 재검토하고, 요청 메소드와 URL의 정확성을 확인</strong>했습니다. 또한, <strong>서버 로그를 분석</strong>하여 실패한 요청을 파악했습니다.<br>

해결책으로는 클라이언트 코드에서 메소드를 명확히 정의하고, <strong>@RequestBody를 반드시 붙이는 형태</strong>로 API 요청을 수정했습니다. 이 과정을 통해 요청 본체가 <strong>올바르게 서버로 전달</strong>되도록 하여 오류를 성공적으로 해결할 수 있었습니다.<br><br>

<h4>📌 FormData를 통한 비동기 데이터 전송의 이해</h4>

비동기 처리 과정에서 <strong>FormData의 데이터 이동</strong>에 대한 이해가 부족하여 어려움을 겪었습니다. 초기에는 클라이언트 측에서 <strong>FormData를 생성하고 서버로 전송하는 과정</strong>이 어떻게 이루어지는지 명확히 파악하지 못했습니다. 특히, 비동기 요청을 통해 데이터를 전송할 때, 데이터가 <strong>어떻게 직렬화되고 전송</strong>되는지에 대한 이해가 부족했습니다.

그러나 여러 자료를 참고하고 <strong>실습을 통해</strong>, FormData가 어떻게 생성되고, Fetch API를 통해 <strong>비동기적으로 서버에 전송되는지를 명확히 이해</strong>하게 되었습니다. FormData 객체는 폼 필드의 값을 키-값 쌍으로 구성하여 전송할 수 있으며, 이는 파일 업로드와 같은 다양한 데이터 유형을 지원하는 데 유용하다는 점을 알게 되었습니다.

이러한 과정을 통해 비동기 요청에서 <strong>FormData의 역할과 데이터 이동 과정을 명확히 이해</strong>하게 되었고, 이를 바탕으로 프로젝트에서의 데이터 처리 및 전송 과정을 효율적으로 개선할 수 있었습니다.

<h3>6-2. 총평</h3>
<h4>🌟 기획: 유지·보수를 생각하자!  </h4>
저번 비워드림 프로젝트 때 느꼈던 점을 되짚어가며 이번 프로젝트 때에는 서버 작업을 할 때 화면 및 DB를 신경쓰지 않을 수 있도록 최대한 꼼꼼하게 기획을 했다. 그러나 직접 프로젝트를 진행하다보니, 사용자의 편의를 위한 기획은 끝이 없다는 것을 깨닫게 되었다. 
<br><br>'완성'하는 것이 아니라 <strong>계속해서 발전해나가고 수정</strong>하는 것이며, 왜 개발자들이 유지 보수를 위한 코드를 강조했는지 깨닫게 되었다.
<br><br>
 <h4>🌟 협업: <strong>프로젝트는 '내'가 아니라 '팀'이 하는 것</strong> </h4>
이번 프로젝트에서 특히 느낀 점은 개인의 작업도 중요하지만, <strong>팀원들의 작업 또한 그만큼 중요</strong>하다는 것입니다. 테이블 컬럼이나 DTO를 수정할 때, 팀원들에게 사전 알림 없이 혼자 진행하는 것은 절대 불가하다는 것을 깨달았습니다. 
코드가 공유되기 때문에 한쪽에서 변경을 하면 다른 팀원의 코드에 영향을 미쳐 오류가 발생할 수 있습니다.
<br> <br>
예를 들어, 로그인 담당자가 member 테이블의 컬럼을 수정했지만, 그 사실이 공유되지 않아 로그인 시 갑작스러운 오류가 발생한 적이 있었습니다. 만약 사전에 상황을 보고했다면 이러한 문제는 발생하지 않았을 것입니다. 이 사건 이후로는 변경 사항에 대해 서로 공유하며 진행하기로 했습니다.
<br><br>
이 프로젝트를 통해, 나의 작업이 다른 팀원에게 미치는 영향을 인지하게 되었고, <strong>팀워크의 중요성</strong>을 다시 한 번 깨닫게 되었습니다.

<br><br>
<h4>🌟 미래: 성장하는 개발자! </h4>
이번 프로젝트를 시작하며 세운 목표는 <strong>'어려운 부분 피하지 않기'</strong>strong>였습니다. 프로젝트를 진행하는 동안 페이징 처리, 첨부파일 관리, 결제 API 통합 등 이전 프로젝트에서 경험하지 못했던 <strong>다양한 도전에 직면</strong>strong>하게 되었습니다.<br><br>

이러한 새로운 과제들을 <strong>성장하고 배운다는 마음가짐</strong>으로 하나씩 해결해 나갔습니다. 각 문제에 부딪힐 때마다 관련 자료를 깊이 있게 학습하고, 필요할 경우 팀원들에게 질문하여 <strong>지식을 확장</strong>했습니다. 이를 통해 서비스의 기능을 하나씩 완성해 나가면서 <strong>큰 자부심과 성취감</strong>을 느꼈습니다.<br><br>

특히, 어려운 문제를 해결했을 때의 <strong>기쁨</strong>은 그 어떤 보상보다 값진 경험이었습니다. 앞으로도 이러한 도전 과제를 통해 계속 성장하고, 더 나은 개발자로 발전해 나가고 싶습니다. 이번 프로젝트는 저에게 단순한 기술적 성장을 넘어, <strong>문제 해결능력과 팀워크의 중요성</strong>을 깨닫게 해주었습니다. 미래에도 이러한 경험을 바탕으로 <strong>끊임없이 배우고 성장하는 개발자</strong>가 돼야겠다고 다짐했습니다.
